# General Ledger (App 210) ‚Äî Functions List

User-facing functions by area. Each area = a `view()` section in `Jet/index.html`; each bullet is a
public method on the single KO viewModel in `Jet/js/app.js`. Mandatory maintenance artifact ‚Äî
update on any view/method/endpoint change.

## Overview (`view()==='overview'`)
- coverage stats: `combinationCount`, `pctClassified`, per-dimension `valueCount` / `mappedSegments` (from `/boot`).

## Classifications (`view()==='classifications'`) ‚Äî manage Sector/Chapter/Program master values
- `clsType` ‚Äî dimension selector (drives `loadValues`).
- `loadValues()` ‚Äî list values for the selected dimension.
- `addValue()` / `editValue(r)` / `saveValue()` ‚Äî create/update (code, EN/AR name, **3 alt names**, tag, parent, order, active).
- `deleteValue(r)` ‚Äî delete (blocked with a message if assignments reference it).
- **Assignments drawer (row click)** ‚Äî `openClsDrill(v)`: clicking a value row opens an extra-wide (1120px) right-edge drawer (`.dw-drawer.dw-xw`) listing **every segment mapped to that value** (`GET /mappings?type=&valueid=` ‚Äî segment code, description, editable start/end dates + notes, CURRENT/PAST chip). Top-right buttons: **+ Add** (`clsAddRow` ‚Äî new row with a datalist segment picker from `/segments/:key/values`, description auto-fills), **Close** (`closeClsDrill`), **Save** (`clsSaveAll` ‚Äî POSTs new rows / PUTs dirty rows sequentially; overlap 400s surface in the drawer error bar; refreshes the values list + counts). Per-row ‚úï = `clsRemoveRow` (DELETE for saved rows, confirm-guarded). Edit/Delete row buttons use `clickBubble:false` so they don't trigger the drill.

## Segment Mapping (`view()==='mapping'`) ‚Äî date-tracked assignments
- `mapType` ‚Äî dimension; `segSearch` + `segOptions` + `mapSegment` ‚Äî real-data segment-value picker.
- `loadSegOptions()` / `loadMappings()` ‚Äî picker + effective-history timeline (current + past).
- `addMapping()` / `editMapping(r)` / `saveMapping()` ‚Äî create/update (value, start/end date, notes); overlap rejected server-side (toast).
- `deleteMapping(r)` ‚Äî remove an assignment.

## Explorer (`view()==='explorer'`) ‚Äî the unified COA view
- filters: `expSearch`, `fSector`, `fChapter`, `asOf` (date) ‚Üí `loadCombos(offset)` (server-paginated).
- `comboRange` ‚Äî ‚Äúa‚Äìb / total‚Äù; `exportCsv()` ‚Äî CSV of the current filtered set (‚â§500 rows).

## Actuals (`view()==='actuals'`) ‚Äî Budget vs Actual report (YTD per GL combination)
- full-viewport width (the `.wrap.wide` modifier is applied on Actuals + Dashboard).
- search criteria: `acPeriod` (**mandatory**, defaults to current period), `acSector`, `acChapter`, `acProgram`, `acAppr`, `acAccount`, `acCostCenter`, `acSource` (transaction-source ‚Äî keep rows whose measure is non-zero; reactive list `acSources` = budget/commitment/obligation/**openCommitment**/**openObligation**/glActual/grn/apDirect), `acSearch` ‚Äî populated by `loadAcFilters()` (`/actuals/filters`, now also returns `accounts` + `costCenters` LOVs).
- `runActuals(offset)` ‚Äî run the report (`/actuals`), server-paginated; `acReset()` ‚Äî clear filters, keep current period; `btnSearch`/`btnReset` buttons + Enter-to-search.
- **Report-parameters drawer (v1.23.0)** ‚Äî the filter card moved OFF the page into a right-edge slide-in drawer (`.dw-filter`, min(720px, 94vw)): `openAcFilters()`/`closeAcFilters()`/`applyAcFilters()` (Apply = search + close; Reset re-runs and keeps the drawer open; Esc closes), page-head **Filters** button with an active-criteria count badge (`acActiveCount`, `.fbadge`), and an **applied-criteria chip bar** on the page (`.ac-chipbar` ‚Äî period chip + one labelled chip per active filter via `acFilterChips`; click anywhere on it to open the drawer).
- **Grouped KPI band (v1.23.0)** ‚Äî the 14 flat `.kstat` tiles became **6 measure-group cards** (`.kgrid`/`.kg-*`): Budget (+ `acUtilTxt`/`acPctW` utilization bar), Commitment (PR) = Total + Open/Pipeline rows, Obligation (PO) = Total + Open/Pipeline rows, Open Encumbrance = total + Open PR/Open PO rows, GL Actual = total + GRN/AP Direct/SLA rows, Funds Available (GL) = total + Calc row + `acFundsTxt` remaining-% (red `.neg` when negative). Each card: accent spine + icon chip + Fraunces value + sub-rows.
- business-question answer cards (14): `tot(k)` reads the `totals` summary ‚Äî budget, **Total PR** (`prTotal`), **Open Commitment** (`openCommitment`), **Commitment Pipeline** (`commitmentPipeline`), **Total PO** (`totalPo`), **Open Obligation** (`openObligation`), **PO Pipeline** (`poPipeline`), **Open Encumbrance** (`openEncumbrance`), GL actual, GRN actual, AP direct, SLA actual, **Funds Available (GL)** (`fundsAvailable`), **Funds Available (calc)** (`fundsAvailableCalc`).
- result table ‚Äî **grouped cells** (`.grp-cell` w/ 3 `.grp-row` each drillable): **Commitment (PR)** = Total / Open / Pipeline (count on Total row); **Obligation (PO)** = Total / Open / Pipeline; **Funds Available** = GL / Calc. Single columns: budget, **Open Encumbrance**, GL actual, GRN, AP direct, SLA. Commitment sourced from real requisitions (`ATD_PR_*`): **Total** = FUNDS_STATUS Reserved+Liquidated / **Open** = Reserved / **Pipeline** = Not-reserved. Obligation from `po_distributions`: **Total** = all except Failed/Passed / **Open** = (Reserved+Partially-Liquidated) **GRN-netted** / **Pipeline** = Failed/Passed. **Open Encumbrance** = Open Commitment + Open Obligation. **AP Direct** = AP lines with `po_number IS NULL`; **SLA** = GRN + AP direct. Group headers carry a ‚ìò hint (`hCommitmentGrp`/`hObligationGrp`/`hFundsGrp`/`hOpenEncumbrance`).
- drill-down: `openDrill(row, metric)` (`/actuals/lines`, metric ‚àà budget|commitment|opencommitment|commitmentpipeline|obligation|openobligation|popipeline|glactual|grn|apdirect) ‚Üí wide modal; `closeDrill()`. **commitment/openCommitment/commitmentPipeline** ‚Üí real PR lines (PR #, description, requester, funds status); **obligation/poPipeline** ‚Üí PO lines; **openObligation** ‚Üí PO lines with Ordered / Received (GRN) / Open columns (the netting made visible). Modal header carries an **Export CSV** button (`drillExportCsv()`, shown once lines are loaded).
- `acExportCsv()` ‚Äî CSV of the full filtered set (‚â§5000 rows, incl. all 3-figure Commitment/Obligation + Open Encumbrance + both Funds Available); `acRange` ‚Äî ‚Äúa‚Äìb of total‚Äù.

## Budget Utilization (`view()==='butil'`) ‚Äî project budget vs actual (per task & expenditure type)
- full-viewport width (`.wrap.wide` also applied on this view). One row per **Budget Year √ó Project √ó Task √ó Expenditure Type**, budget > 0 lines only.
- page layout (v1.14.0) = three **brand-headed regions** (`.bu-sec`): **Search** (filters, collapsible), **Overview** (KPI band, collapsible), **Results** (table + pager, maximizable). `toggleBuSec(k)` collapses with a rotating chevron; state persists in `localStorage('gl_bu_ui')`; collapsed headers show live summaries (`buSearchSummary` = year + active-filter count via `buActiveFilters`; `buKpiSummary` = compact Budget ¬∑ Fund). `toggleBuMax()`/`buMax` maximizes the Results region to a full-screen surface (z 55, below the drill drawer/modal/toast so drills open above it; sticky thead + pinned pager; body scroll locked; **Esc restores**, guarded while a drawer/modal is open).
- search criteria: `buYear` (**mandatory**, defaults to latest year), `buPeriod` (**Accounting period, v1.16.0** ‚Äî Full year + the 12 MM-YYYY months of the selected year via `buPeriodOpts`; **YTD semantics** with a ‚ìò hint; defaults to the **current period** when the year is current ‚Äî first load, Reset and year-change), `buType` (project type, **defaults to `DCT OPEX Project Type`** when present in the LOV), `buSector`, plus four **autocomplete LOV filters** (v1.12.0, native `<datalist>`): `buCc` (Cost Center ‚Äî option label = department), `buProject` (Project ‚Äî option value = number, label = name; contains-match on number OR name), `buTask` (Task), `buEtype` (Expenditure Type), and free-text `buSearch`. Select LOVs from `loadBuFilters()` (`/butil/filters`); autocomplete lists from `loadBuLovs()` (`/butil/lov?year=`, reloaded when `buYear` changes).
- `runButil(offset)` ‚Äî run the report (`/butil`, server-paginated); `buReset()` (clears filters + all multi-select chips, restores the DCT OPEX default type); `btnSearch`/`btnReset` + Enter-to-search on every filter input; `yearRequired` toast when no year.
- **Multi-select Chapter / Cost centre / Project (v1.23.0)** ‚Äî every Chapter pick (`buChapterPick`/`buChapterAdd`) and every exact Cost-centre/Project LOV pick (`buCcCommit`/`buProjCommit` on the datalist inputs' change/Enter) becomes a **chip** in a labelled bar under the filter grid (`.mchips-bar`/`.mchip`, √ó removes via `buChipRemove`); `buChapterSel`/`buCcSel`/`buProjSel` observableArrays. Requests send the chips **pipe-delimited** (`buChapterParam()`/`buCcParam()`/`buProjParam()` ‚Äî a pipe list = exact any-of match server-side; a single free-typed pipe-less value keeps the original contains-match). Flows into `/butil`, the pager, CSV export, both drill modes AND the Briefing Book.
- **"Figures in" display-unit selector (v1.24.0)** ‚Äî Search-region field (`buUnit`: Auto B/M/K | Billions | Millions | Thousands | Exact number, i18n `fUnitL`/`unit*`) driving the unit-aware formatter `buNum(n)` used by the 4 KPI tiles, the duo breakdown rows, the collapsed-band summary (`buKpiSummary`) and all 6 results-table money columns. Display-only (instant re-render, no re-query); persisted in `localStorage('gl_bu_ui')`; exact tooltips + raw CSV unchanged.
- loading states (v1.15.0): while `/butil` runs the results tbody shows **8 shimmer skeleton rows** (`skRows`/`skCols`, `.sk-row`/`.sk`), an indeterminate `.ldbar` sweep under the Results header, and the Search button goes `.busy`; every other load slot app-wide (Actuals/Dashboard/Classifications/Explorer/drill drawers/modal) uses the **branded `.ldr` loader** (GL diamond in twin counter-rotating arcs + i18n `loadingData` text).
- totals **answer band** (**4 tiles since v1.23.0** ‚Äî `.bu-kpis`/`.bk-*`, scoped to this view) via `buTot(k)`: **Budget** | **Total Actual** (`buActualTot` = actualAp + actualGrn) | **Total Encumbrance** (`buEncumbTot` = commitmentPr [open PR] + obligationPo [open PO, GRN-netted]) | **Fund Available** (= Budget ‚àí AP ‚àí GRN ‚àí Open PR ‚àí Open PO). The two composite tiles (`.bk-duo`) show a **stacked composition bar** (`.bk-seg`, `buSegW(part,total)`) + a `buTotPctTxt` %-of-budget line + two **drillable breakdown rows** (`.bk-duo-row`: AP/GRN and PR/PO with colour dots and per-component amounts + a bracketed **%-of-budget suffix** per row via `buPartPct(v)` [v1.23.1] ‚Äî click drills that component via `openBuAgg`). Each tile = per-measure gradient tint + accent spine + icon chip + Fraunces value + **%-of-budget context line** + **mini utilization bar** (`buPct(k)`/`buBarW(k)`/`buPctTxt(k)`; Budget tile shows the filtered row count, Fund tile uses `buFundSub` ‚Äî "% of budget remaining", or red `.neg` "over budget" when negative). The 4 measure tiles (`.bk-click`) drill to all supporting lines; responsive 6‚Üí3‚Üí2‚Üí1 columns.
- result table (17 cols): Type / Sector / Department / Cost centre / Project (number + name) / Task / GL Account / Appropriation / Chapter / Program / Expenditure type + the 6 measures. Attributes resolve task-first (task `COST_CENTER`‚ÜíSECTOR map, task `APPROPRIATION`‚ÜíCHAPTER map, task `PROGRAM`), then transactions, then project.
- `buExportCsv()` ‚Äî CSV of the full filtered set (‚â§5000 rows); `buRange` ‚Äî ‚Äúa‚Äìb of total‚Äù.
- drill-down (two entry points, one **right-edge slide-in drawer** ‚Äî `.dw-*`, mirrors the platform edit-drawer; brand-themed header + eyebrow + context; `closeDrawer()`):
  - `openBuDrill(row, metric)` ‚Äî the four money **table cells** (`.money-cell`) drill that single budget line (`/butil/lines?year=&project=&task=&etype=&metric=`); eyebrow = project¬∑name, context = task¬∑expenditure type.
  - `openBuAgg(metric)` ‚Äî the four **KPI cards** (`.kstat-click`: Actual AP / Actual GRN / Commitment PR / Obligation PO) drill **all** supporting lines across the current filter set (`/butil/lines?year=&metric=&projecttype=&sector=&search=&costcenter=&fproject=&ftask=&fetype=` ‚Äî the LOV filters travel f-prefixed so they don't collide with the row-drill key params); eyebrow = "All budget lines ¬∑ <year>", context = active filters, adds Project/Task columns, and a `drillCapNote` "showing top N of M" when capped at 1000.
  - metric ‚àà ap|grn|pr|po; totals always reconcile: **ap**‚ÜíAP invoices (invoice #, date, vendor, currency, invoice amount, distribution AED, validation, payment, description), **grn**‚ÜíGRN receipts (receipt #, date, currency, rate, amount AED; rows sorted by GRN # then line ‚Äî cap selection stays top-by-amount), **pr**‚Üíopen-commitment PR lines (PR #, description, budget date, currency, line amount AED), **po**‚Üíopen-obligation PO lines GRN-netted (PO #, line, budget date, vendor, funds status, PO amount, line amount AED, Open AED). Reuses the shared drill observables (`drillCols`/`drillRows`/`drillTotalV`/`drillCount`/`cell`).
  - `drillExportCsv()` (v1.11.0) ‚Äî **Export CSV** button in the drawer header (and the Actuals drill-modal header): serializes the loaded `drillCols`/`drillRows` (server-driven columns, raw values, quote-escaped) + a Total reconciliation footer row; UTF-8 BOM for Excel-safe Arabic; filename `gl_drill_<title>_<sub>.csv`; hidden while loading / when no lines.
  - `drillLink(row, col)` (v1.20.0) ‚Äî **Fusion deep-links** in the drawer: Invoice/PO/PR numbers render as new-tab anchors (`.fus-link`, ‚Üó) via the SHARED `shared/js/fusionLinks.js` (UMD ‚Üí `window.FusionLinks`; loaded by index.html before app.js) when the row carries the FUSION internal id ‚Äî `invoiceId` (ap), `poHeaderId` (grn/po), `prHeaderId` (pr) from `/butil/lines`; plain text otherwise. CSV export keeps numbers only.
- `runBuXlsx()` ‚Äî **Export Excel** button (page head, v1.30.0, internal analysis): generates the Reporting-Platform `BUDGET_UTIL_REGISTER` workbook (reporting/db/25 ‚Äî 6 worksheets: utilization lines, direct AP invoices, GRN receipts, open purchase orders, open requisitions, pending-approval PR/PO queue) with the page's full filter set; `POST /butil/xlsx` ‚Üí 5-s poll ‚Üí auto-download `Budget_Utilization_Register_<year>.xlsx`.
- `runBuBook()` ‚Äî **Briefing Book** button (page head; **v1.22.0: sends ALL page filters**; **v1.30.0: the book gained Part 5 "Pending Approvals ‚Äî PR & PO Queue"** + an Approval-queue insight, Observations renumbered Part 6): generates the Reporting-Platform `BUDGET_UTIL_BOOK` PDF for the page's **full filter set** ‚Äî Year, Period (YTD), Project type, Sector, Chapter, Cost center, Project, Task, Expenditure type, Search ‚Äî with the page's exact predicate semantics, so the book scope always matches the page (the cover prints every applied filter). Flow: `POST /butil/book` (GL-gated bridge, `11_gl_butil_book_ords.sql`) ‚Üí toast "queued ‚Äî run #N" ‚Üí 6s polling of `GET /butil/book/:id` (‚â§ ~6 min) ‚Üí authed blob download of `GET /butil/book/:id/pdf` as `Budget_Utilization_Briefing_Book_<year>.pdf`; `buBookBusy` drives the "Preparing book‚Ä¶" label. The run is rendered by the normal reporting worker fleet (also runnable from the BI app).

## Projects Encumbrances (`view()==='encumbrances'`) ‚Äî open PO/PR encumbrance lines, full GL combination (v1.25.0)
- **"Open Projects Encumbrance Follow-up"** ‚Äî every OPEN encumbrance line (Open Commitment PR + Open Obligation PO, GRN-netted) for the budget lines matching the Budget Utilization scope, exploded to the FULL GL combination (all ten segments, **code + name**) plus sector/chapter names. Rendered in the **SHARED `<interactive-report>` component** (the same APEX-IR grid AP/BI use): column show/hide/reorder/rename, filter chips, multi-sort, calculated columns, aggregates, control breaks, highlight rules, CSV + SheetJS XLSX export, and per-user localStorage layouts (`reportCode:'GL_PROJ_ENCUMBRANCES'`, `layoutsApi:null`).
- **Search criteria are the SAME as Budget Utilization** ‚Äî the tab reuses the `bu*` filter observables (`buParams`) and the shared `bu-*` autocomplete datalists verbatim (Year req, Period YTD, Project type, Sector, Chapter[multi], Cost centre[multi], Project[multi], Task, Expenditure type, Search); the Search button runs this tab.
- `runEncumbrances()` ‚Äî one-shot capped fetch (`GET /encumbrances`, limit 10 000) ‚Üí builds the IR envelope (`enData`) from the server's `columns[]`+`items[]`. `enCount` / `enOpenTotal` (+ `enOpenTotalTxt`) / `enTruncated` drive the summary line; the **Open total reconciles to the butil Total Encumbrance** (Commitment PR + Obligation PO). Auto-runs on first open (via `go('encumbrances')`), re-runs on Search.
- `enIrError` ‚Äî set from `window._irLoadError` if the shared component failed to load (index.html requirejs boot); the tab then shows a graceful notice instead of the grid.
- IR enablement: GL (a portal-style app) bootstraps requirejs in `index.html` (shim `knockout`‚Üíthe existing global, `shared`/`text`/`xlsx` paths), requires `shared/components/interactiveReport` (+`shared/editDrawer`, `shared/i18n`) so the `<interactive-report>`/`<edit-drawer>` components register on the app's KO before `app.js` boots; `toggleLang` also flips `shared/i18n` so the grid re-renders in AR. `platform.css` is linked **before** `app.css` (portal styles still win) to supply the IR base classes; the IR var bridge in `app.css` themes the grid green.

## Encumbrances ‚Äì Pending Approval (`view()==='pending'`) ‚Äî PR/PO documents awaiting approval (v1.27.0)
- **"Encumbrances ‚Äì Pending Approval Follow-up"** ‚Äî every PR / PO document currently PENDING APPROVAL in Fusion (daily BIP snapshot `atd_pr_po_pending_approval`, otbi-atd/db/47) joined to the open-encumbrance line detail (project / task / expenditure type, budget date, AED amounts, full 10-segment GL combination code+name) PLUS the approval trail (preparer/buyer, submitted date, pending-with approver, days pending, business unit, funds status). Data = `PROD.DCT_PR_PO_PENDING_V` (db/v2/52). **Zero-value lines are excluded (v1.30.1)** ‚Äî the page follows budget utilization; a zero line never reserves funds (the reserved/not-reserved split stays visible; book + Excel register are additionally reserved-only).
- **Search criteria are the SAME as Budget Utilization** ‚Äî reuses the `bu*` filter observables + shared `bu-*` datalists verbatim; the Search/Reset buttons run this tab (`runPending()`).
- **Monitoring KPI band** (server-aggregated over the FULL filtered set, correct even when the register is capped): Pending documents (PR/PO split + seg bar) ¬∑ Pending amount AED (PR/PO split) ¬∑ **Funds reserved vs not-reserved** (reserved = already inside the butil open-encumbrance figures; not-reserved = would hit Fund Available on approval) ¬∑ **Approval aging** (avg days, oldest, over-30-days docs+amount, red flavour when any are stalled). **Every tile breakdown row DRILLS (v1.30.0)** ‚Äî `openPnDrill('pr'|'po'|'reserved'|'unreserved'|'over30'|'within30')` opens the SHARED right-edge drill drawer with the matching pending lines (client-side slice of `pnItems`, cap 1,000 w/ top-N note, reconciling total, CSV export; Document # cells deep-link to Fusion via the generic `drillLink` docNumber rule).
- **Aging analysis + Top pending approvers mini tables** (`pnAging`/`pnApprovers` ‚Äî buckets 0-7/8-15/16-30/31+ at document grain; top 8 approvers by value held w/ doc count + max days ‚Äî the follow-up chase list). Rows drill too (`openPnBucket`/`openPnApprover`, v1.30.0); headers are **#79C5AC region-header bands** (user-specified).
- **Busy overlay** (v1.30.0) ‚Äî the butil oj-progress-circle replica over the KPI band + Results while `/gl/pending` runs (`pnLoading` on the `.bu-body` wrapper).
- **Extract-coverage note** (`pnUnmatchedTxt`) ‚Äî snapshot documents with no matched project budget line (`in_extract='N'`) are counted, never silently dropped; the Briefing Book annex lists them.
- Register in the **SHARED `<interactive-report>`** (`reportCode:'GL_ENC_PENDING'`, `section:'pend'`, cap 10 000, default sort days-pending desc) with the same GL-combination hover popover as the encumbrances tab; `pnIrError` fallback notice.
- **Source filter** (`pnSource`, v1.28.0) ‚Äî All / PR / PO dropdown in the Search region (page-local, cleared by Reset; `source=` on `/pending`) + **"Showing figures in"** (the shared `buUnit` selector ‚Äî KPI tiles and mini-table amounts format via `buNum`).
- **Business Unit multi-select** (`pnBus`/`pnBuSel`/`pnBuAdd`, v1.31.0) ‚Äî dropdown fed by the `businessUnits[]` LOV in the `/pending` response; each pick becomes a chip on the shared mchips bar (Reset clears). Sends `bu=` pipe-delimited EXACT any-of list, scoping the register, every KPI/aging/approver aggregate AND the unmatched extract-coverage KPI; forwarded to both the Briefing Book and the Excel register. The BIP snapshot is the platform's only cross-BU source (the OTBI extracts are DCT-scoped), which is why the filter lives on this page.
- **Fusion deep-links on Document #** (`pnGridOver`/`pnGridClick`, v1.28.0) ‚Äî hovering a docNumber cell shows the link affordance (`.pn-doclink` + tooltip), clicking opens the Fusion PR/PO deep link in a new tab via the shared `fusionLinks.js` builders over `fusionHeaderId` (kept in a VM side-map keyed source|doc# ‚Äî the shared IR grid drops undeclared row fields). Same delegated ko.contextFor pattern as the combination popover; no shared-component change.
- `runPnBook()` ‚Äî **Briefing Book** button ‚Üí `POST /pending/book` with the full page filter set, 6-s poll (`GET /pending/book/:id`), auto-download of the PDF (`ENC_PENDING_BOOK`, reporting/db/23 ‚Äî **funds-reserved non-zero lines only**, registers carry Sector/Cost centre/Appropriation code+name, full untruncated text).
- `runPnXlsx()` ‚Äî **Export Excel** button (v1.29.0, internal analysis) ‚Üí `POST /pending/xlsx` with the full page filter set + source, 4-s poll (`GET /pending/xlsx/:id`), auto-download of `Encumbrances_Pending_Approval_Register_<year>.xlsx` (`ENC_PENDING_REGISTER`, reporting/db/24 ‚Äî sheet 1 = the 29-column flat pending PR/PO register on the book scope rule; sheet 2 = the extract-coverage annex).

## Refresh (Overview + Actuals + Dashboard) & Rebuild (Overview + Budget Utilization)
- `refreshActuals()` ‚Äî `POST /actuals/refresh` rebuilds `DCT_GL_COA_SNAP` so the report reflects the latest GL/ATD data + mapping edits; `refreshing` / `lastRefreshed`. Also runs hourly (`DCT_ACTUALS_REFRESH_JOB`, db/v2/35) and is mirrored as a button in the ATD app (`POST /atd/actuals/refresh`).
- `rebuildViews()` ‚Äî **Rebuild views** button (Overview + Budget Utilization page heads, tooltip hint + confirm): `POST /actuals/rebuild` ‚Üí `prod.dct_views_rebuild` (db/v2/38). For **structural** reloads (new/renamed columns): re-creates the 16 `SELECT *` base pass-through views over the `ATD_*` tables, recompiles INVALIDs, refreshes the snapshot; toast reports any views still invalid (those need a script edit). For a plain data reload, Refresh actuals is enough.
