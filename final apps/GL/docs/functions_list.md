# General Ledger (App 210) — Functions List

User-facing functions by area. Each area = a `view()` section in `Jet/index.html`; each bullet is a
public method on the single KO viewModel in `Jet/js/app.js`. Mandatory maintenance artifact —
update on any view/method/endpoint change.

## Overview (`view()==='overview'`)
- coverage stats: `combinationCount`, `pctClassified`, per-dimension `valueCount` / `mappedSegments` (from `/boot`).

## Classifications (`view()==='classifications'`) — manage Sector/Chapter/Program master values
- `clsType` — dimension selector (drives `loadValues`).
- `loadValues()` — list values for the selected dimension.
- `addValue()` / `editValue(r)` / `saveValue()` — create/update (code, EN/AR name, **3 alt names**, tag, parent, order, active).
- `deleteValue(r)` — delete (blocked with a message if assignments reference it).
- **Assignments drawer (row click)** — `openClsDrill(v)`: clicking a value row opens an extra-wide (1120px) right-edge drawer (`.dw-drawer.dw-xw`) listing **every segment mapped to that value** (`GET /mappings?type=&valueid=` — segment code, description, editable start/end dates + notes, CURRENT/PAST chip). Top-right buttons: **+ Add** (`clsAddRow` — new row with a datalist segment picker from `/segments/:key/values`, description auto-fills), **Close** (`closeClsDrill`), **Save** (`clsSaveAll` — POSTs new rows / PUTs dirty rows sequentially; overlap 400s surface in the drawer error bar; refreshes the values list + counts). Per-row ✕ = `clsRemoveRow` (DELETE for saved rows, confirm-guarded). Edit/Delete row buttons use `clickBubble:false` so they don't trigger the drill.

## Segment Mapping (`view()==='mapping'`) — date-tracked assignments
- `mapType` — dimension; `segSearch` + `segOptions` + `mapSegment` — real-data segment-value picker.
- `loadSegOptions()` / `loadMappings()` — picker + effective-history timeline (current + past).
- `addMapping()` / `editMapping(r)` / `saveMapping()` — create/update (value, start/end date, notes); overlap rejected server-side (toast).
- `deleteMapping(r)` — remove an assignment.

## Explorer (`view()==='explorer'`) — the unified COA view
- filters: `expSearch`, `fSector`, `fChapter`, `asOf` (date) → `loadCombos(offset)` (server-paginated).
- `comboRange` — “a–b / total”; `exportCsv()` — CSV of the current filtered set (≤500 rows).

## Actuals (`view()==='actuals'`) — Budget vs Actual report (YTD per GL combination)
- full-viewport width (the `.wrap.wide` modifier is applied on Actuals + Dashboard).
- search criteria: `acPeriod` (**mandatory**, defaults to current period), `acSector`, `acChapter`, `acProgram`, `acAppr`, `acAccount`, `acCostCenter`, `acSource` (transaction-source — keep rows whose measure is non-zero; reactive list `acSources` = budget/commitment/obligation/**openCommitment**/**openObligation**/glActual/grn/apDirect), `acSearch` — populated by `loadAcFilters()` (`/actuals/filters`, now also returns `accounts` + `costCenters` LOVs).
- `runActuals(offset)` — run the report (`/actuals`), server-paginated; `acReset()` — clear filters, keep current period; `btnSearch`/`btnReset` buttons + Enter-to-search.
- **Report-parameters drawer (v1.23.0)** — the filter card moved OFF the page into a right-edge slide-in drawer (`.dw-filter`, min(720px, 94vw)): `openAcFilters()`/`closeAcFilters()`/`applyAcFilters()` (Apply = search + close; Reset re-runs and keeps the drawer open; Esc closes), page-head **Filters** button with an active-criteria count badge (`acActiveCount`, `.fbadge`), and an **applied-criteria chip bar** on the page (`.ac-chipbar` — period chip + one labelled chip per active filter via `acFilterChips`; click anywhere on it to open the drawer).
- **Grouped KPI band (v1.23.0)** — the 14 flat `.kstat` tiles became **6 measure-group cards** (`.kgrid`/`.kg-*`): Budget (+ `acUtilTxt`/`acPctW` utilization bar), Commitment (PR) = Total + Open/Pipeline rows, Obligation (PO) = Total + Open/Pipeline rows, Open Encumbrance = total + Open PR/Open PO rows, GL Actual = total + GRN/AP Direct/SLA rows, Funds Available (GL) = total + Calc row + `acFundsTxt` remaining-% (red `.neg` when negative). Each card: accent spine + icon chip + Fraunces value + sub-rows.
- business-question answer cards (14): `tot(k)` reads the `totals` summary — budget, **Total PR** (`prTotal`), **Open Commitment** (`openCommitment`), **Commitment Pipeline** (`commitmentPipeline`), **Total PO** (`totalPo`), **Open Obligation** (`openObligation`), **PO Pipeline** (`poPipeline`), **Open Encumbrance** (`openEncumbrance`), GL actual, GRN actual, AP direct, SLA actual, **Funds Available (GL)** (`fundsAvailable`), **Funds Available (calc)** (`fundsAvailableCalc`).
- result table — **grouped cells** (`.grp-cell` w/ 3 `.grp-row` each drillable): **Commitment (PR)** = Total / Open / Pipeline (count on Total row); **Obligation (PO)** = Total / Open / Pipeline; **Funds Available** = GL / Calc. Single columns: budget, **Open Encumbrance**, GL actual, GRN, AP direct, SLA. Commitment sourced from real requisitions (`ATD_PR_*`): **Total** = FUNDS_STATUS Reserved+Liquidated / **Open** = Reserved / **Pipeline** = Not-reserved. Obligation from `po_distributions`: **Total** = all except Failed/Passed / **Open** = (Reserved+Partially-Liquidated) **GRN-netted** / **Pipeline** = Failed/Passed. **Open Encumbrance** = Open Commitment + Open Obligation. **AP Direct** = AP lines with `po_number IS NULL`; **SLA** = GRN + AP direct. Group headers carry a ⓘ hint (`hCommitmentGrp`/`hObligationGrp`/`hFundsGrp`/`hOpenEncumbrance`).
- drill-down: `openDrill(row, metric)` (`/actuals/lines`, metric ∈ budget|commitment|opencommitment|commitmentpipeline|obligation|openobligation|popipeline|glactual|grn|apdirect) → wide modal; `closeDrill()`. **commitment/openCommitment/commitmentPipeline** → real PR lines (PR #, description, requester, funds status); **obligation/poPipeline** → PO lines; **openObligation** → PO lines with Ordered / Received (GRN) / Open columns (the netting made visible). Modal header carries an **Export CSV** button (`drillExportCsv()`, shown once lines are loaded).
- `acExportCsv()` — CSV of the full filtered set (≤5000 rows, incl. all 3-figure Commitment/Obligation + Open Encumbrance + both Funds Available); `acRange` — “a–b of total”.

## Budget Utilization (`view()==='butil'`) — project budget vs actual (per task & expenditure type)
- full-viewport width (`.wrap.wide` also applied on this view). One row per **Budget Year × Project × Task × Expenditure Type**, budget > 0 lines only.
- page layout (v1.14.0) = three **brand-headed regions** (`.bu-sec`): **Search** (filters, collapsible), **Overview** (KPI band, collapsible), **Results** (table + pager, maximizable). `toggleBuSec(k)` collapses with a rotating chevron; state persists in `localStorage('gl_bu_ui')`; collapsed headers show live summaries (`buSearchSummary` = year + active-filter count via `buActiveFilters`; `buKpiSummary` = compact Budget · Fund). `toggleBuMax()`/`buMax` maximizes the Results region to a full-screen surface (z 55, below the drill drawer/modal/toast so drills open above it; sticky thead + pinned pager; body scroll locked; **Esc restores**, guarded while a drawer/modal is open).
- search criteria: `buYear` (**mandatory**, defaults to latest year), `buPeriod` (**Accounting period, v1.16.0** — Full year + the 12 MM-YYYY months of the selected year via `buPeriodOpts`; **YTD semantics** with a ⓘ hint; defaults to the **current period** when the year is current — first load, Reset and year-change), `buType` (project type, **defaults to `DCT OPEX Project Type`** when present in the LOV), `buSector`, plus four **autocomplete LOV filters** (v1.12.0, native `<datalist>`): `buCc` (Cost Center — option label = department), `buProject` (Project — option value = number, label = name; contains-match on number OR name), `buTask` (Task), `buEtype` (Expenditure Type), and free-text `buSearch`. Select LOVs from `loadBuFilters()` (`/butil/filters`); autocomplete lists from `loadBuLovs()` (`/butil/lov?year=`, reloaded when `buYear` changes).
- `runButil(offset)` — run the report (`/butil`, server-paginated); `buReset()` (clears filters + all multi-select chips, restores the DCT OPEX default type); `btnSearch`/`btnReset` + Enter-to-search on every filter input; `yearRequired` toast when no year.
- **Multi-select Chapter / Cost centre / Project (v1.23.0)** — every Chapter pick (`buChapterPick`/`buChapterAdd`) and every exact Cost-centre/Project LOV pick (`buCcCommit`/`buProjCommit` on the datalist inputs' change/Enter) becomes a **chip** in a labelled bar under the filter grid (`.mchips-bar`/`.mchip`, × removes via `buChipRemove`); `buChapterSel`/`buCcSel`/`buProjSel` observableArrays. Requests send the chips **pipe-delimited** (`buChapterParam()`/`buCcParam()`/`buProjParam()` — a pipe list = exact any-of match server-side; a single free-typed pipe-less value keeps the original contains-match). Flows into `/butil`, the pager, CSV export, both drill modes AND the Briefing Book.
- **"Figures in" display-unit selector (v1.24.0)** — Search-region field (`buUnit`: Auto B/M/K | Billions | Millions | Thousands | Exact number, i18n `fUnitL`/`unit*`) driving the unit-aware formatter `buNum(n)` used by the 4 KPI tiles, the duo breakdown rows, the collapsed-band summary (`buKpiSummary`) and all 6 results-table money columns. Display-only (instant re-render, no re-query); persisted in `localStorage('gl_bu_ui')`; exact tooltips + raw CSV unchanged.
- loading states (v1.15.0): while `/butil` runs the results tbody shows **8 shimmer skeleton rows** (`skRows`/`skCols`, `.sk-row`/`.sk`), an indeterminate `.ldbar` sweep under the Results header, and the Search button goes `.busy`; every other load slot app-wide (Actuals/Dashboard/Classifications/Explorer/drill drawers/modal) uses the **branded `.ldr` loader** (GL diamond in twin counter-rotating arcs + i18n `loadingData` text).
- totals **answer band** (**4 tiles since v1.23.0** — `.bu-kpis`/`.bk-*`, scoped to this view) via `buTot(k)`: **Budget** | **Total Actual** (`buActualTot` = actualAp + actualGrn) | **Total Encumbrance** (`buEncumbTot` = commitmentPr [open PR] + obligationPo [open PO, GRN-netted]) | **Fund Available** (= Budget − AP − GRN − Open PR − Open PO). The two composite tiles (`.bk-duo`) show a **stacked composition bar** (`.bk-seg`, `buSegW(part,total)`) + a `buTotPctTxt` %-of-budget line + two **drillable breakdown rows** (`.bk-duo-row`: AP/GRN and PR/PO with colour dots and per-component amounts + a bracketed **%-of-budget suffix** per row via `buPartPct(v)` [v1.23.1] — click drills that component via `openBuAgg`). Each tile = per-measure gradient tint + accent spine + icon chip + Fraunces value + **%-of-budget context line** + **mini utilization bar** (`buPct(k)`/`buBarW(k)`/`buPctTxt(k)`; Budget tile shows the filtered row count, Fund tile uses `buFundSub` — "% of budget remaining", or red `.neg` "over budget" when negative). The 4 measure tiles (`.bk-click`) drill to all supporting lines; responsive 6→3→2→1 columns.
- result table (17 cols): Type / Sector / Department / Cost centre / Project (number + name) / Task / GL Account / Appropriation / Chapter / Program / Expenditure type + the 6 measures. Attributes resolve task-first (task `COST_CENTER`→SECTOR map, task `APPROPRIATION`→CHAPTER map, task `PROGRAM`), then transactions, then project.
- `buExportCsv()` — CSV of the full filtered set (≤5000 rows); `buRange` — “a–b of total”.
- drill-down (two entry points, one **right-edge slide-in drawer** — `.dw-*`, mirrors the platform edit-drawer; brand-themed header + eyebrow + context; `closeDrawer()`):
  - `openBuDrill(row, metric)` — the four money **table cells** (`.money-cell`) drill that single budget line (`/butil/lines?year=&project=&task=&etype=&metric=`); eyebrow = project·name, context = task·expenditure type.
  - `openBuAgg(metric)` — the four **KPI cards** (`.kstat-click`: Actual AP / Actual GRN / Commitment PR / Obligation PO) drill **all** supporting lines across the current filter set (`/butil/lines?year=&metric=&projecttype=&sector=&search=&costcenter=&fproject=&ftask=&fetype=` — the LOV filters travel f-prefixed so they don't collide with the row-drill key params); eyebrow = "All budget lines · <year>", context = active filters, adds Project/Task columns, and a `drillCapNote` "showing top N of M" when capped at 1000.
  - metric ∈ ap|grn|pr|po; totals always reconcile: **ap**→AP invoices (invoice #, date, vendor, currency, invoice amount, distribution AED, validation, payment, description), **grn**→GRN receipts (receipt #, date, currency, rate, amount AED; rows sorted by GRN # then line — cap selection stays top-by-amount), **pr**→open-commitment PR lines (PR #, description, budget date, currency, line amount AED), **po**→open-obligation PO lines GRN-netted (PO #, line, budget date, vendor, funds status, PO amount, line amount AED, Open AED). Reuses the shared drill observables (`drillCols`/`drillRows`/`drillTotalV`/`drillCount`/`cell`).
  - `drillExportCsv()` (v1.11.0) — **Export CSV** button in the drawer header (and the Actuals drill-modal header): serializes the loaded `drillCols`/`drillRows` (server-driven columns, raw values, quote-escaped) + a Total reconciliation footer row; UTF-8 BOM for Excel-safe Arabic; filename `gl_drill_<title>_<sub>.csv`; hidden while loading / when no lines.
  - `drillLink(row, col)` (v1.20.0) — **Fusion deep-links** in the drawer: Invoice/PO/PR numbers render as new-tab anchors (`.fus-link`, ↗) via the SHARED `shared/js/fusionLinks.js` (UMD → `window.FusionLinks`; loaded by index.html before app.js) when the row carries the FUSION internal id — `invoiceId` (ap), `poHeaderId` (grn/po), `prHeaderId` (pr) from `/butil/lines`; plain text otherwise. CSV export keeps numbers only.
- `runBuBook()` — **Briefing Book** button (page head; **v1.22.0: sends ALL page filters**): generates the Reporting-Platform `BUDGET_UTIL_BOOK` PDF for the page's **full filter set** — Year, Period (YTD), Project type, Sector, Chapter, Cost center, Project, Task, Expenditure type, Search — with the page's exact predicate semantics, so the book scope always matches the page (the cover prints every applied filter). Flow: `POST /butil/book` (GL-gated bridge, `11_gl_butil_book_ords.sql`) → toast "queued — run #N" → 6s polling of `GET /butil/book/:id` (≤ ~6 min) → authed blob download of `GET /butil/book/:id/pdf` as `Budget_Utilization_Briefing_Book_<year>.pdf`; `buBookBusy` drives the "Preparing book…" label. The run is rendered by the normal reporting worker fleet (also runnable from the BI app).

## Projects Encumbrances (`view()==='encumbrances'`) — open PO/PR encumbrance lines, full GL combination (v1.25.0)
- **"Open Projects Encumbrance Follow-up"** — every OPEN encumbrance line (Open Commitment PR + Open Obligation PO, GRN-netted) for the budget lines matching the Budget Utilization scope, exploded to the FULL GL combination (all ten segments, **code + name**) plus sector/chapter names. Rendered in the **SHARED `<interactive-report>` component** (the same APEX-IR grid AP/BI use): column show/hide/reorder/rename, filter chips, multi-sort, calculated columns, aggregates, control breaks, highlight rules, CSV + SheetJS XLSX export, and per-user localStorage layouts (`reportCode:'GL_PROJ_ENCUMBRANCES'`, `layoutsApi:null`).
- **Search criteria are the SAME as Budget Utilization** — the tab reuses the `bu*` filter observables (`buParams`) and the shared `bu-*` autocomplete datalists verbatim (Year req, Period YTD, Project type, Sector, Chapter[multi], Cost centre[multi], Project[multi], Task, Expenditure type, Search); the Search button runs this tab.
- `runEncumbrances()` — one-shot capped fetch (`GET /encumbrances`, limit 10 000) → builds the IR envelope (`enData`) from the server's `columns[]`+`items[]`. `enCount` / `enOpenTotal` (+ `enOpenTotalTxt`) / `enTruncated` drive the summary line; the **Open total reconciles to the butil Total Encumbrance** (Commitment PR + Obligation PO). Auto-runs on first open (via `go('encumbrances')`), re-runs on Search.
- `enIrError` — set from `window._irLoadError` if the shared component failed to load (index.html requirejs boot); the tab then shows a graceful notice instead of the grid.
- IR enablement: GL (a portal-style app) bootstraps requirejs in `index.html` (shim `knockout`→the existing global, `shared`/`text`/`xlsx` paths), requires `shared/components/interactiveReport` (+`shared/editDrawer`, `shared/i18n`) so the `<interactive-report>`/`<edit-drawer>` components register on the app's KO before `app.js` boots; `toggleLang` also flips `shared/i18n` so the grid re-renders in AR. `platform.css` is linked **before** `app.css` (portal styles still win) to supply the IR base classes; the IR var bridge in `app.css` themes the grid green.

## Dashboard (`view()==='dashboard'`) — executive analytics
- `dashPeriod` selector → `loadDashboard()` (`/dashboard`); KPI strip via `kpis()` (budget, actual, funds, encumbrance, PO total & count, utilisation/commitment %).
- `gauge()` — utilisation radial gauge (SVG); `trend()` — period-over-period actual SVG area+line; `sectorBars`/`programBars`/`apprBars` — horizontal bar charts (actual / PO commitments); `insights()` — auto-generated executive sentences. All charts are hand-built SVG/CSS (no chart library).

## Refresh (Overview + Actuals + Dashboard) & Rebuild (Overview + Budget Utilization)
- `refreshActuals()` — `POST /actuals/refresh` rebuilds `DCT_GL_COA_SNAP` so the report reflects the latest GL/ATD data + mapping edits; `refreshing` / `lastRefreshed`. Also runs hourly (`DCT_ACTUALS_REFRESH_JOB`, db/v2/35) and is mirrored as a button in the ATD app (`POST /atd/actuals/refresh`).
- `rebuildViews()` — **Rebuild views** button (Overview + Budget Utilization page heads, tooltip hint + confirm): `POST /actuals/rebuild` → `prod.dct_views_rebuild` (db/v2/38). For **structural** reloads (new/renamed columns): re-creates the 16 `SELECT *` base pass-through views over the `ATD_*` tables, recompiles INVALIDs, refreshes the snapshot; toast reports any views still invalid (those need a script edit). For a plain data reload, Refresh actuals is enough.
