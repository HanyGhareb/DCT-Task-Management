Overview

This document outlines the full project plan to rebuild the i-Finance Task Management web application using Oracle technologies hosted on Oracle Cloud Infrastructure (OCI) Free Tier. The application enables a Finance Director to monitor weekly tasks assigned to five section managers, with drill-down views, file attachments, charts, and real-time cross-user data visibility.

Target Architecture

Browser (Oracle JET App) ↕ HTTPS Nginx (Reverse Proxy + SSL) — OCI VM (Ubuntu 22.04) ↕ ORDS (Oracle REST Data Services) — OCI VM ↕ JWT Bearer Token validation OCI IAM (Identity Domain) — Authentication & Authorization ↕ Oracle 23ai Free Tier DB — OCI Autonomous Database

Phase 1
OCI Infrastructure Setup

Phase 2
OCI IAM Setup
Goal: Configure identity, users, roles, and OAuth2 token flow — estimated 1 day
#TaskDetail
2.1Create IAM Identity DomainUse Default domain or create a dedicated domain for the app
2.2Create IAM GroupsCreate groups: ifinance-directors, ifinance-managers
2.3Create IAM UsersOne user per team member — assign each to the appropriate group

Phase 3
Database Schema
Goal: Design and create all database objects — estimated 1 day
#TaskDetail
3.1Create DB schema/userCreate IFINANCE schema with appropriate grants and quotas
3.2Create USERS tableMirror IAM users — store section, role, display name, color/avatar preferences
3.3Create TASKS tableAll task fields: title, description, status, priority, category, dates, completion %, next action, notes, week number, year
3.4Create TASK_ATTACHMENTS tableFile name, MIME type, file size, BLOB content — foreign key to TASKS
3.5Create TASK_TAGS tableMany-to-many tags per task
3.6Create sequences & triggersAuto-increment PKs; CREATED_AT / UPDATED_AT timestamp triggers
3.7Create indexesIndexes on ASSIGNED_TO, WEEK_NUMBER, YEAR, STATUS for query performance
3.8Create TASK_WEEKLY_STATS viewPre-aggregated completion stats per manager per week
3.9Create MANAGER_OVERVIEW viewDirector-level cross-section summary — used by KPI cards
3.10Seed reference dataInsert users (synced with IAM), task categories, status and priority constants

Phase 5
ORDS REST API
Goal: Expose all DB operations as secured REST endpoints — estimated 1.5 days
#TaskDetail
5.1REST-enable IFINANCE schemaRun ORDS.ENABLE_SCHEMA — configure URL prefix
5.2GET /tasksReturn tasks for current user filtered by week — identity from JWT claim
5.3POST /tasksCreate new task — validate required fields
5.4PUT /tasks/:idUpdate existing task — enforce ownership check
5.5DELETE /tasks/:idDelete task — enforce ownership check
5.6GET /tasks/:idSingle task full detail including attachments list
5.7GET /stats/weeklyManager's own weekly stats — completion rate, by-status counts
5.8GET /stats/overviewDirector-only endpoint — all managers summary for KPI cards
5.9GET /stats/trendCompletion trend over last N weeks — used by trend chart
5.10POST /attachmentsUpload file as multipart/form-data — store as BLOB
5.11GET /attachments/:idDownload file — stream BLOB with correct MIME type
5.12DELETE /attachments/:idRemove attachment — enforce task ownership
5.13GET /users/meReturn current user profile resolved from JWT claims
5.14Apply IAM JWT validationConfigure ORDS privilege and role mapping for all endpoints
5.15Test all endpoints with PostmanCover authenticated, unauthenticated, and unauthorized cases
